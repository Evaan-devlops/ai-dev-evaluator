# AI Dev Evaluator — Development Rules

## Deployment Constraints (MANDATORY)

These rules apply to ALL development work in this project:

- **Local only**: Everything must be developed locally on this machine only.
- **No remote GitHub**: Do not create, modify, or push any remote GitHub repository.
- **No git push / gh commands**: Do not run `git push`, `gh repo create`, `gh auth`, or any GitHub sync command.
- **No pull requests**: Do not open pull requests.
- **No publishing**: Do not publish anything to any external service.
- **Local project folder only**: Work only in the local project folder.
- **Local git OK**: It is fine to initialize a local git repo only if needed, but do not connect any remote.
- **No remote changes**: If git is already initialized, do not add or change remotes.
- **Local file changes only**: Show changes locally in files only.
- **Ask before destructive commands**: Ask before running any destructive command (e.g., `rm -rf`, `git reset --hard`, dropping data).
